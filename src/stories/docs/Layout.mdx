# Layout
Компоненты для построения макета страниц

#### Импорт компонентов
```tsx
import { Layout } from 'isp-ui-kit'
const { Column, NotFoundPage } = Layout
```

## Интерфейс "Три колонки"
Компоненты для построения колончатого интерфейса
### [Column](?path=/story/layout-threecolumns-column--empty)
- [Empty](?path=/story/layout-threecolumns-column--empty): Базовый пример компонента Column без содержимого. Подходит для демонстрации структуры или настройки макета.
- [With Elements](?path=/story/layout-threecolumns-column--with-elements): Пример компонента Column с элементами. Подходит для отображения данных или других вложенных компонентов.

---

### [EmptyData](?path=/story/layout-threecolumns-emptydata--example)
Компонент, используемый для отображения пустых данных. Применяется для сценариев, когда нет данных для отображения.

---

### [NoData](?path=/story/layout-threecolumns-nodata--example)
Вариация компонента для работы с отсутствующими данными. Полезен для создания пользовательских сообщений или заглушек.

---

## Страницы
Набор шаблонных страниц для базовых сценариев

### [ErrorPage](?path=/story/layout-pages-errorpage--example)
Компонент для отображения страницы ошибки. Используется для обработки ошибок 404, 500 или других случаев, требующих пользовательского сообщения.

---

### [HomePage](?path=/story/layout-pages-homepage--example)
Главная страница, используемая для отображения основного контента приложения или приветственного экрана.

---

### [NotFoundPage](?path=/story/layout-pages-notfoundpage--example)
Компонент для отображения страницы "Страница не найдена". Применяется для обработки запросов к несуществующим маршрутам.

---
